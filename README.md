# 健康管理系统

## 项目简介

健康管理系统是一个专为个人健康数据追踪和分析设计的单页应用，帮助用户记录、分析和管理各项健康指标，包括体重、体脂率、肌肉量、蛋白质等，通过数据可视化和风险等级评估，为用户提供直观的健康状况分析。

## 功能特点

### 核心功能

1. **健康数据录入**：支持录入体重、体脂率、肌肉量、水分率、蛋白质、骨量、内脏脂肪等多项健康指标
2. **数据可视化**：通过ECharts图表展示各项健康指标的变化趋势
3. **健康数据比对**：支持选择两个不同日期的数据进行详细比对
4. **风险等级评估**：自动计算各项指标的风险等级，包括BMI、血压、体脂率、蛋白质等
5. **每日健康数据**：点击图表数据点查看详细的每日健康数据
6. **个人基础信息管理**：记录身高、年龄、性别等基础信息，用于计算衍生指标
7. **数据导入导出**：支持JSON格式的数据导入导出功能

### 指标分析

- **体重**：记录体重变化，计算体重变化百分比
- **体脂率**：评估体脂率水平，提供健康建议
- **BMI**：根据身高体重计算BMI值，评估体重状态
- **肌肉量/肌肉率**：追踪肌肉量变化，评估肌肉健康状况
- **蛋白质**：评估蛋白质水平，划分为不足、标准、优三个等级
- **水分率**：监测身体水分含量
- **内脏脂肪**：评估内脏脂肪等级
- **骨量**：记录骨量变化
- **血压**：评估血压风险等级
- **肥胖度**：计算肥胖度百分比，评估肥胖程度

## 技术栈

- **前端框架**：原生HTML5 + CSS3 + JavaScript
- **图表库**：ECharts 5.4.3
- **数据存储**：localStorage
- **数据格式**：JSON
- **开发工具**：任何文本编辑器

## 快速开始

### 方法一：直接打开

1. 下载或克隆项目到本地
2. 直接在浏览器中打开 `index.html` 文件
3. 系统会自动从 `data.json` 加载初始数据

### 方法二：本地服务器（推荐）

1. 下载或克隆项目到本地
2. 在项目目录启动本地服务器
   ```bash
   # 使用Python 3
   python -m http.server 8000
   
   # 或使用Python 2
   python -m SimpleHTTPServer 8000
   
   # 或使用Node.js
   npx http-server -p 8000
   ```
3. 在浏览器中访问 `http://localhost:8000`

## 核心功能使用指南

### 1. 健康数据录入

1. 点击顶部导航栏的"设置"按钮
2. 选择"健康数据录入"选项
3. 在弹出的模态框中填写各项指标
4. 点击"添加/更新数据"按钮保存

### 2. 查看健康数据趋势

- 主页面的图表展示了体重和体脂率的变化趋势
- 可以通过"图表指标选择"区域勾选其他指标进行查看
- 点击图表中的数据点可以查看详细的每日健康数据

### 3. 健康数据比对

1. 点击顶部导航栏的"设置"按钮
2. 选择"健康数据比对"选项
3. 在弹出的模态框中选择两个不同的日期
4. 点击"开始比对"按钮查看详细对比结果

### 4. 个人基础信息管理

1. 点击顶部导航栏的"设置"按钮
2. 选择"个人基础信息"选项
3. 在弹出的模态框中填写身高、年龄、性别等信息
4. 点击"保存基础信息"按钮保存

## 风险等级评估标准

### BMI等级
- 偏瘦：< 18.5
- 正常：18.5 - 23.9
- 超重：24.0 - 27.9
- 肥胖：≥ 28.0

### 蛋白质等级
- 不足：< 16.0%
- 标准：16.0% - 20.0%
- 优：> 20.0%

### 血压等级
- 正常：收缩压 < 120 且 舒张压 < 80
- 高血压前期：收缩压 120-139 或 舒张压 80-89
- 高血压1级：收缩压 140-159 或 舒张压 90-99
- 高血压2级：收缩压 ≥ 160 或 舒张压 ≥ 100

### 体脂率等级
- **男性**：偏瘦 < 10%，正常 10%-20%，超重 20%-25%，肥胖 > 25%
- **女性**：偏瘦 < 15%，正常 15%-25%，超重 25%-30%，肥胖 > 30%

### 肥胖度等级
- 消瘦：< -20.0%
- 偏瘦：-20.0% - -10.0%
- 标准：-10.0% - 10.0%
- 偏胖：10.0% - 20.0%
- 肥胖：20.0% - 50.0%
- 重度：> 50.0%

## 技术实现

### 数据管理

- 使用localStorage存储用户数据，确保刷新页面后数据不丢失
- 支持从data.json文件加载初始数据
- 提供数据导入导出功能，方便数据备份和迁移

### 图表实现

- 使用ECharts 5.4.3实现数据可视化
- 支持多指标同时展示，通过Y轴分组显示不同单位的指标
- 支持图表缩放、数据点点击查看详情等交互功能

### 响应式设计

- 适配不同屏幕尺寸，在移动设备上也能正常使用
- 模态框采用响应式布局，确保在小屏幕上也能完整显示
